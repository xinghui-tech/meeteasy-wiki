# 插件管理

Admin 中的插件管理用于安装、配置全局插件，并按租户控制启用范围。

## 插件类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Embedded | Python 包，运行在后端进程内 | 内置 AI、OCR、报名增强 |
| Sidecar | 独立 HTTP 服务 | 第三方集成、定制业务 |

开发者文档见 [插件开发](/developer/plugins)。

## 查看已安装插件

1. 登录 Admin → **插件管理**。
2. 列表展示插件名称、版本、状态（已启用/已停用）。
3. 点击插件名称进入详情，查看配置项与依赖说明。

## 配置插件

1. 在插件详情页填写必要参数（如 AI 提供商 API Key、模型名称、超时时间）。
2. **敏感信息**（密钥）仅平台管理员可见，存储于服务端加密配置。
3. 保存后，配置对启用该插件的租户生效。

::: tip 提示
API Key 请勿写入 Console 或 MeetApp 前端；所有密钥由 Admin 统一配置。
:::

## 按租户启用

1. 在插件详情 → **租户范围** 中选择启用租户。
2. 未启用的租户在 Console 中看不到对应能力（如 AI 助手入口）。
3. 可随时停用某租户的插件，不影响其他租户。

## 常见问题

**Q：插件更新后需要重启吗？**  
A：Embedded 插件通常需重启 API 服务；Sidecar 插件可独立滚动更新。详见 [部署运维](/developer/deployment)。

**Q：租户能否自行安装插件？**  
A：默认由平台 Admin 统一管理；定制插件需通过商务与研发评估后上架。
