# 签到管理

Console 签到模块用于现场核验参会者身份、统计到场人数，并与票码、座位联动。

## 签到方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 扫码签到 | 扫描参会者票码 QR | 签到处固定设备 |
| 自助签到 | MeetApp 内一键签到 | 参会者自带手机 |
| 手动签到 | Console 搜索姓名/手机标记 | 补录、VIP 通道 |

## 配置签到

1. 会议详情 → **签到设置**。
2. 启用签到，设置 **签到开放时间**（如会议开始前 2 小时）。
3. 选择允许的签到方式。
4. 可选：签到成功页文案、是否显示座位信息。

## 现场操作（Console）

### 扫码签到

1. 打开 Console **签到** 页 → **扫码模式**。
2. 对准参会者 MeetApp 票码或 PDF 票券二维码。
3. 验证通过后显示姓名与座位（如已分配）。

### 手动查询

1. **签到列表** → 搜索姓名/手机/票码。
2. 点击 **标记已签到**；误操作可撤销（需权限）。

## 统计与导出

- 实时显示：应到/已到/未到人数。
- 按时间段查看签到高峰。
- 导出签到记录供后勤与餐饮对接。

## 与座位模块联动

若已配置 [座位管理](/user-manual/console/seating)，签到成功页可展示 **桌号/座位号**，引导就座。

## 常见问题

**Q：未报名能否现场签到？**  
A：默认需有效票码；组织者可在 Console 现场 **补录报名** 后签到（视权限配置）。

**Q：重复签到如何处理？**  
A：系统识别已签到状态，提示「已签到」并不重复计数。
